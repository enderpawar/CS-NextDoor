  경쟁 공백 분석

  기존 경쟁 툴(HWiNFO, CrystalDiskInfo, PC-Doctor)의 공통 한계:

  - 숫자는 보여주지만 해석은 사용자 몫 ("CPU 92%가 나쁜 건가요?"에 아무도 답 안 함)
  - 전부 영어 전용 + 전문가 용어
  - PC가 켜진 상태 전제 — 부팅 불가 시나리오 대응 없음
  - 카메라·마이크 입력 지원 제품: 0개
  - AI 자연어 진단 탑재 제품: 0개

  결론: "비전문가 대상 + AI 해석 + 카메라/마이크 + 한국어 + 부팅 불가 대응"을 동시에 만족하는 제품이
   현재 시장에 없다.

  예상 사용자 반응

  얼리어답터 (출시 0~3개월)

  - 퀘이사존·클리앙에서 자발적 리뷰 예상
  - "비프음을 마이크로 인식"이 강한 화제성 유발 → 유튜브 소재로 적합
  - 정확도 검증 추적이 동반됨 — 초반 진단 결과가 실제로 맞아야 신뢰 확보

  일반 사용자 (3~12개월, 주요 성장 동력)

  - "컴퓨터 비프음 뭐야", "BSOD 해결법" 검색 유입이 핵심
  - PWA 특성상 설치 없이 접근 → 전환 장벽 낮음
  - "수리기사 필요 없음" 결론이 나올 때 강한 만족 + 공유 발생
  - "수리기사 권장" 결론을 "돈 낭비를 막아줬다"로 프레이밍하는 온보딩이 필수

  핵심 리스크 3가지

  1. API 비용 (최고 위험) — IP 기반 일 5회 쿼터 미설정 시 공개 직후 비용 폭증. 이것이 서비스 생존의 
  첫 번째 조건
  2. 첫 오진 사례 — "AI가 RAM 문제라 했는데 실제로는 파워 문제였다"는 사례가 커뮤니티에 퍼지면 신뢰 
  회복이 어려움. 확신도 표시 + 면책 고지 UI가 완충재
  3. 데이터 프라이버시 — 시스템 스냅샷·이벤트 로그를 외부 AI 서버로 전송하는 구조. 진단 전 동의 UI가
   이탈 지점이 될 수 있음

  ---
  수익화 권장 로드맵

  출시~6개월:  완전 무료 (일 5회 쿼터) → 사용자·정확도 데이터 축적
  6~18개월:   프리미엄 월 3,900원 (무제한 진단 + 이력 저장)
  18개월 이후: B2B 팀 플랜 (PC방·중소기업 IT팀)
  +α:         "수리기사 권장" 화면 → 신뢰 수리점 연결 제휴 수수료
              "RAM 교체 필요" → 다나와/쿠팡 부품 링크 제휴 커미션

  전략적 제언

  1. 마케팅 첫 메시지: "PC가 안 켜질 때도 스마트폰으로 진단 가능" — 경쟁사 어디에도 없는 유일한 강점
  2. SEO 우선: 앱스토어보다 "PC 이상한 소리", "컴퓨터 비프음", "BSOD 원인" 한국어 롱테일 키워드 검색
   유입이 핵심 채널
  3. 초기 집중: 비프음 인식 + BSOD 분석 두 사용 사례에 집중해 정확도를 빠르게 높이는 것이 "만능 앱" 
  표방보다 PMF 달성에 유리
  4. Phase 10 피드백 루프를 빨리: 진단 정확도 통계를 투명하게 공개("X건 진단, 만족도 Y%")하면 신뢰  
  구축 속도가 빨라짐
